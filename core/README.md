# CODEX Core

CODEX Core contains normative requirements shared by every conforming CODEX project.

## Planned modules

1. Scope and terminology.
2. Conformance model.
3. Document classes and lifecycle.
4. Source and edition provenance.
5. Semantic content model.
6. Metadata requirements.
7. Persistent identifiers and references.
8. Revision and decision history.
9. Accessibility requirements.
10. Publication and preservation requirements.
11. Responsible automation and AI requirements.
12. Validation and release requirements.

## Status labels

- **Proposal** — early concept;
- **Draft** — active development;
- **Review** — ready for structured review;
- **Accepted** — approved normative text;
- **Deprecated** — retained for compatibility;
- **Archived** — no longer maintained.

No file in this directory is normative merely because it exists. Normative status must be declared in its metadata and release manifest.
