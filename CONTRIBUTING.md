# Contributing to CODEX

## Before contributing

Read `MANIFESTO.md`, `CHARTER.md`, `ARCHITECTURE.md`, and `GOVERNANCE.md`.

## Contribution paths

- report a problem through an issue;
- propose a small correction through a pull request;
- propose a substantial change through an RFC;
- record an architectural decision through an ADR;
- contribute a guide, template, schema, test, or example.

## Required qualities

Contributions should be:

- self-contained and understandable outside chat history;
- explicit about normative versus non-normative content;
- traceable to evidence or clearly labeled as design proposals;
- written in accessible language;
- compatible with long-term preservation goals;
- free of copied copyrighted material unless permission and attribution are documented.

## Pull requests

A pull request should describe:

1. the problem;
2. the proposed solution;
3. affected documents;
4. compatibility implications;
5. alternatives considered;
6. validation performed.

## Commit messages

Use concise conventional prefixes where practical:

- `docs:` documentation;
- `spec:` normative specification;
- `feat:` new capability;
- `fix:` correction;
- `schema:` machine-readable schema;
- `build:` publication tooling;
- `chore:` maintenance.

## AI-assisted contributions

AI assistance must be disclosed when it materially shaped normative text, translation, factual commentary, or code. Human contributors remain responsible for verification, rights, accuracy, and final approval.
