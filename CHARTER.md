# CODEX Project Charter

**Version:** 0.0.1  
**Status:** Draft  
**Project:** CODEX — Open Architecture for Digital Scholarly Editions

## Mission

Create an open, durable, and reproducible framework for scholarly digital editions that combines editorial rigor, transparent provenance, accessible publication, and long-term preservation.

## Objectives

- define CODEX Core requirements;
- provide reference architectures and implementation guidance;
- publish reusable templates, schemas, checklists, and examples;
- support multiple output formats from one master source;
- establish transparent governance and versioning;
- validate the framework through reference projects.

## Non-goals for Genesis

Genesis does not attempt to prescribe one mandatory software stack, replace expert review, certify scholarly truth, or define every domain-specific editorial practice.

## Authority and accountability

Repository maintainers approve releases and normative changes. Contributors may propose changes through issues, RFCs, ADRs, and pull requests. Published claims remain accountable to identifiable human editors.

## Document classes

- **CORE:** binding requirements of CODEX Core;
- **SPEC:** technical or editorial specifications;
- **RFC:** proposed changes or extensions;
- **ADR:** accepted architectural decisions;
- **GUIDE:** non-normative implementation guidance;
- **TEMPLATE:** reusable project artifacts;
- **EXAMPLE:** reference implementations.

## Lifecycle

Idea → Draft → Review → Accepted → Maintained → Deprecated → Archived.

## Success criteria for Core 1.0

- coherent normative core;
- documented governance;
- metadata and content model;
- editorial and translation standards;
- accessibility and preservation requirements;
- reference templates;
- tested multi-format publication workflow;
- at least one reference project profile.
