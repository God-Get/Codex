# CODEX Vision

## Long-term vision

CODEX is an open, reproducible architecture for preparing, maintaining, publishing, and preserving scholarly editions in multiple human- and machine-readable formats.

A CODEX project should be able to outlive any individual contributor, software stack, publishing platform, or AI model.

## Intended users

- translators;
- textual scholars and philologists;
- historians and philosophers;
- editors and publishers;
- librarians and archivists;
- software developers;
- research groups and educational institutions;
- independent scholarly projects.

## Desired outcomes

A conforming project should support:

1. a single authoritative master source;
2. explicit separation of source, translation, commentary, and interpretation;
3. persistent identifiers and cross-references;
4. versioned editorial decisions;
5. export to accessible digital and print formats;
6. durable storage in open formats;
7. independent review and reproducible publication;
8. responsible use of automation and AI;
9. migration to future tools without loss of knowledge.

## Scope of CODEX Core

CODEX Core defines the minimum governance, editorial, structural, metadata, quality, and preservation requirements shared by all profiles and projects.

Specialized profiles may extend Core for poetry, parallel texts, manuscript traditions, critical apparatus, sacred texts, dictionaries, or other domains.

## First reference project

HERMETICA is planned as the first practical implementation of CODEX for a multilingual, annotated edition of classical Hermetic texts.
